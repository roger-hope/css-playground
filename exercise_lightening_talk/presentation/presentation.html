Tree-structural Pseudo-classes

These pseudo-classes relate to the position of an element within the document tree.

  • :first-child
  • :nth-child()
  • :only-child
/* Tree-structural pseudo-classes */
td:first-child{ background-color: palevioletred; }
td:last-child{ background-color: palevioletred; }
tr:nth-child(even) { background-color: #8fa7bd; }
tr:nth-child(odd) { background-color: #c0ac66; }

User Action Pseudo-classes

These pseudo-classes apply styles based on user interactions like hovering or clicking.

  • :hover
  • :active

/* User action pseudo-classes */

.joinBtn { width: 10em; height: 5ex; background-color: gold; border: 2px solid firebrick; border-radius: 10px; font-weight: bold; color: black; cursor: pointer; }

/*hover part css*/

.joinBtn:hover { background-color: red; }

/*active part css*/

.joinBtn:active { box-shadow: 2px 2px 5px red; border: 5px blue solid; }

/* Focus part css*/

label { display: block; margin-top: 1em; }

input:focus { background-color: red; }

select:focus { background-color: blue; }

Functional Pseudo-classes

These pseudo-classes take arguments and provide more complex matching capabilities.

  • :is()
  • :where()
  • :has()
  • :not()

/* Functional pseudo-classes */

:is(header, footer) {background-color: #333;color: white;}

:where(.hero, .features) {margin-bottom: 2em;}
